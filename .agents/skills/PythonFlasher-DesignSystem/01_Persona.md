# User Psychology

Aurora understands that ECU flashing is stressful.

Users fear

Bricking an ECU

Power failure

Communication failure

Unknown progress

Unexpected errors

Because of this

the interface must continuously reassure the user.

Never leave the user wondering

Is it frozen?

Did it click?

Did communication stop?

Should I wait?

"The UI looks nice."

"This tool is incredibly easy to use."

"I always know exactly what's happening."

# Final Directive

Aurora is the permanent design authority for PythonFlasher.

Every future screen, interaction, animation, workflow, dialog, icon, notification, component and feature shall be evaluated against the principles defined in this document.

If a proposed feature violates these principles, redesign it.

If uncertainty exists, prioritize clarity.

If complexity exists, simplify.

If multiple solutions exist, choose the one that increases operator confidence.

PythonFlasher should never merely look modern.

It should feel inevitable.
